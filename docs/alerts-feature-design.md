# DriveWeb 告警中心功能文档

## 1. 文档目的
本文档用于说明 DriveWeb 告警中心模块的页面功能、交互流程与展示规则，作为产品验收与后续迭代依据。

## 2. 功能范围
截至 2026-04-24，已实现：
1. 告警中心列表页（筛选、分页、跳转详情、行内快捷处置）。
2. 告警详情页（基本信息、风险指标、事件上下文、处置操作区、处置记录时间轴）。
3. 告警确认 / 误报 / 关闭弹窗与提交流程。
4. 左侧导航“告警中心”入口。

截至 2026-04-24，未实现：
1. WebSocket 实时推送驱动刷新。

## 3. 角色与权限
| 角色 | 页面可见性 | 当前可执行动作 |
|---|---|---|
| ADMIN | 可访问列表与详情 | 查询、查看详情、确认、标记误报、关闭 |
| OPERATOR | 可访问列表与详情 | 查询、查看详情、确认、标记误报、关闭 |
| VIEWER | 可访问列表与详情 | 查询、查看详情 |

说明：
1. `ADMIN` 和 `OPERATOR` 在满足状态机条件时展示处置按钮。
2. `VIEWER` 不展示列表页和详情页处置入口。

## 4. 页面路由
- 告警列表页：`/alerts`
- 告警详情页：`/alerts/:id`

## 5. 告警列表页功能
### 5.1 筛选区
支持条件：
1. 车队（`fleetId`）
2. 车辆（`vehicleId`）
3. 司机（`driverId`）
4. 风险等级（`riskLevel`：1低/2中/3高）
5. 告警状态（`status`：0新建/1确认/2误报/3关闭）
6. 触发时间范围（`startTime`、`endTime`）

交互规则：
1. 页面首次进入且 URL 未携带时间参数时，默认查询最近 24 小时。
2. 首次默认查询会向后端传递 `startTime` 和 `endTime`。
3. 点击“筛选查询”后按最新条件查询。
4. 点击“重置条件”后恢复默认筛选并重新查询。

### 5.2 列表区
表格字段：
1. 告警编号
2. 车辆
3. 司机
4. 风险级别
5. 疲劳分
6. 分心分
7. 状态
8. 触发时间
9. 操作（仅 `ADMIN` / `OPERATOR` 可见）

交互规则：
1. 点击行或告警编号进入详情页。
2. 列表为空时展示空态文案“暂无告警数据”。
3. 风险级别与状态按标签颜色区分。
4. `ADMIN` / `OPERATOR` 可在行内直接发起处置动作。
5. 列表行内处置按钮按状态机展示：
   `NEW` 展示“确认告警 / 标记误报 / 关闭告警”；
   `CONFIRMED` 展示“标记误报 / 关闭告警”；
   `FALSE_POSITIVE` 与 `CLOSED` 不展示可执行操作。
6. 点击处置按钮会打开对应弹窗，提交成功后刷新当前列表页数据。

### 5.3 分页区
功能：
1. 支持页码切换。
2. 支持每页条数切换（10/20/50/100）。
3. 切换每页条数后回到第一页。

## 6. 告警详情页功能
### 6.1 基本信息卡
展示字段：
1. 告警ID
2. 告警编号
3. 风险等级
4. 当前状态
5. 触发时间

### 6.2 风险指标卡
展示字段：
1. `fatigueScore`
2. `distractionScore`
3. `perclos`
4. `blinkRate`
5. `yawnCount`

### 6.3 事件上下文
展示字段：
1. 车队
2. 车辆
3. 司机
4. 算法版本
5. 头姿态
6. 事件时间

### 6.4 处置记录时间轴
功能说明：
1. 展示后端返回的处置记录（按时间倒序）。
2. 若无处置记录，展示空态“暂无处置记录”。
3. 单条记录展示动作名称、操作人、备注。

### 6.5 处置操作区
功能说明：
1. 仅 `ADMIN` / `OPERATOR` 可见。
2. 按当前告警状态展示可执行动作按钮。
3. 点击动作按钮后弹出对应处置弹窗。
4. 提交成功后刷新当前详情，确保状态与时间轴同步更新。
5. 若当前状态已不可继续流转，则展示“当前状态不可继续处置”。

### 6.6 页面操作
1. 返回列表：返回上一页或兜底跳转 `/alerts`。
2. 刷新：重新拉取当前告警详情。

## 7. 处置弹窗规则
### 7.1 通用规则
1. 弹窗支持备注输入，最大长度 255。
2. 备注允许为空，但输入框会提供动作示例占位文案。
3. 提交成功后关闭弹窗，并提示成功消息。
4. 若服务端返回状态已变化，前端提示“状态已更新，请刷新后重试”，并主动刷新当前页面数据。

### 7.2 弹窗类型
1. 确认告警：提示人工确认风险，备注示例“已电话提醒司机”。
2. 标记误报：提示补充误报原因，备注示例“遮挡导致误报”。
3. 关闭告警：提示补充关闭原因，备注示例“风险已解除”。

## 8. 展示规则
1. 时间字段按浏览器本地时区显示。
2. 分数字段按百分比展示（0~1 范围自动换算为 0%~100%）。
3. 空值统一展示为 `-`。
4. 状态标签文案为：`新建`、`已确认`、`误报`、`已关闭`。

## 9. 状态机展示规则
1. `NEW`：可执行确认、标记误报、关闭。
2. `CONFIRMED`：可执行标记误报、关闭。
3. `FALSE_POSITIVE`：终态，无可执行处置。
4. `CLOSED`：终态，无可执行处置。

## 10. URL 状态保持
列表页会将筛选条件与分页状态同步到 URL query，便于：
1. 页面刷新后恢复当前查询条件。
2. 详情页返回列表后保持上下文。

补充说明：
1. 当前如果 URL 没有任何筛选参数，并不会查询全部告警。
2. 当前实现会补上“最近 24 小时”的默认时间范围后再发起查询。
