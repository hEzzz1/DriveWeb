# DriveWeb Web前端界面功能设计文档

## 1. 文档目的
本文档用于定义 DriveWeb 的前端界面与交互功能设计。
设计依据来自 DriveServer 已定义接口与业务规则，但本文档重点是页面功能本身，不展开后端实现。

参考接口文档：
- `/Users/m1ngyangg/Documents/DriveServer/docs/server-design/03-api-spec.md`

## 2. 设计范围
包含：
1. 登录与鉴权界面。
2. 实时总览界面。
3. 告警列表、详情与处置界面。
4. 趋势统计与风险排行界面。
5. 规则管理界面。
6. 审计与系统状态展示界面。

不包含：
1. 边缘侧设备接入与算法推理界面。
2. 后端模块实现细节。

## 3. 角色与权限视图
| 角色 | 可见页面 | 可执行动作 |
|---|---|---|
| ADMIN | 全部页面 | 全部动作（规则增改启停、告警处置） |
| OPERATOR | 登录、总览、告警、统计、审计 | 告警确认/误报/关闭，不可改规则 |
| VIEWER | 登录、总览、告警只读、统计只读 | 无写操作 |

## 4. 全局交互规范
1. 顶部全局区：系统名称、当前用户、角色标签、退出登录。
2. 左侧导航：总览、告警中心、统计分析、规则管理、审计状态。
3. 响应结构处理：统一解析 `code/message/data/traceId`。
4. 错误提示：错误弹窗显示业务提示 + `traceId`。
5. 时间显示：页面按本地时区展示，筛选传参按接口规范提交 ISO-8601。
6. 状态反馈：所有写操作必须有 `加载中`、`成功`、`失败` 三态。

## 4.1 Web端技术栈（补充）
| 层级 | 技术选型 | 说明 |
|---|---|---|
| 前端框架 | Vue 3 + TypeScript + Vite | 快速开发与类型安全，适合中后台场景 |
| 路由与状态 | Vue Router + Pinia | 支持多页面权限路由与全局状态管理 |
| UI组件 | Element Plus | 表格、表单、弹窗、分页等组件成熟 |
| 图表可视化 | Apache ECharts | 趋势图、排行图、告警分布图能力完整 |
| 数据请求 | Axios | 统一封装鉴权、错误码、traceId 处理 |
| 实时通信 | WebSocket（原生） | 对接 `/ws/alerts` 实时告警推送 |
| 表单校验 | async-validator | 与 Element Plus 配合用于规则配置/处置输入 |
| 权限控制 | JWT + RBAC 路由守卫 | 对齐 `ADMIN/OPERATOR/VIEWER` 权限模型 |
| 质量保障 | ESLint + Prettier + Husky + lint-staged | 统一代码规范与提交前检查 |
| 测试体系 | Vitest + Playwright | 单测 + E2E，覆盖关键页面流程 |
| 部署方式 | Nginx + Docker | 与后端 Stage/Prod-like 部署方式一致 |

## 5. 页面功能设计

### 5.1 登录页
功能目标：完成用户身份认证并建立会话。

页面组件：
1. 用户名输入框。
2. 密码输入框。
3. 登录按钮。
4. 错误提示区域。

交互流程：
1. 用户输入账号密码，点击登录。
2. 调用 `POST /api/v1/auth/login`。
3. 成功后保存 `token`、`expireAt`、`roles` 并跳转总览。
4. 失败时根据 `40101/40301` 展示提示。

接口映射：
- `POST /auth/login`

异常与边界：
1. 输入为空时禁止提交。
2. 接口超时时可重试。
3. token 即将过期时顶部提示续登。

---

### 5.2 实时总览页
功能目标：快速展示当前运营风险态势与实时告警变化。

页面组件：
1. KPI 卡片区：在线车辆数、当前告警车辆数、近5分钟告警量、告警等级分布。
2. 实时告警流：按时间倒序显示新告警。
3. 风险趋势缩略图：疲劳分与分心分近时段走势。
4. 连接状态条：WebSocket 已连接/重连中/降级轮询。

交互流程：
1. 页面加载调用 `GET /api/v1/realtime/overview`。
2. 建立 WebSocket 连接 `/ws/alerts`。
3. 收到 `ALERT_CREATED` 时在告警流首位插入并高亮。
4. 收到 `ALERT_UPDATED` 时同步更新状态。

接口映射：
- `GET /realtime/overview`
- `WS /ws/alerts`

异常与边界：
1. WebSocket 断开后指数退避重连。
2. 连续重连失败时切换短轮询模式并提示。

---

### 5.3 告警中心页（列表）
功能目标：支持运营人员快速筛选、定位、处理告警。

页面组件：
1. 筛选栏：车队、车辆、司机、风险等级、状态、时间范围。
2. 告警表格：告警编号、车辆、司机、风险级别、疲劳分、分心分、状态、触发时间。
3. 批量区（只读可隐藏）：当前版本默认单条处理。
4. 分页器：页码、每页数量。

交互流程：
1. 当前实现默认加载最近 24 小时告警；若 URL 未携带时间参数，前端会自动补充 `startTime` 与 `endTime`。
2. 用户变更筛选项后重新查询。
3. 点击行进入详情页。
4. ADMIN/OPERATOR 可在行内执行快捷操作（确认/误报/关闭）。

接口映射：
- `GET /alerts`
- `POST /alerts/{id}/confirm`
- `POST /alerts/{id}/false-positive`
- `POST /alerts/{id}/close`

异常与边界：
1. 查询结果为空展示空态页。
2. `42901` 时提示请求频繁并自动延迟重试。
3. VIEWER 隐藏处置按钮。

当前实现说明（2026-04-10）：
1. 已实现筛选、分页、列表到详情跳转。
2. 已实现列表筛选条件与分页状态的 URL 同步。
3. 当前如果没有筛选参数，并不会直接查询全部告警，而是默认查询最近 24 小时。
4. 行内处置按钮暂未接入（当前为只读列表）。

---

### 5.4 告警详情页
功能目标：查看单告警的触发上下文和处置轨迹。

页面组件：
1. 基本信息卡：告警编号、触发时间、风险等级、当前状态。
2. 风险指标卡：fatigueScore、distractionScore、perclos、blinkRate、yawnCount。
3. 事件上下文：车辆、司机、算法版本、头姿态。
4. 处置记录时间轴：确认/误报/关闭日志。
5. 处置操作区：确认、误报、关闭按钮 + remark 输入。

交互流程：
1. 页面加载调用详情接口。
2. 根据当前状态与角色控制可执行动作。
3. 提交动作后刷新详情和列表缓存。

接口映射：
- `GET /alerts/{id}`
- `POST /alerts/{id}/confirm`
- `POST /alerts/{id}/false-positive`
- `POST /alerts/{id}/close`

异常与边界：
1. 已关闭告警不可重复关闭。
2. remark 为空时禁止提交。

当前实现说明（2026-04-10）：
1. 已实现基本信息、风险指标、事件上下文、处置记录时间轴展示。
2. 已实现详情页刷新与返回列表。
3. 处置操作区（确认/误报/关闭 + remark）暂未接入。

### 5.5 告警处置弹窗（通用组件）
功能目标：统一处理告警状态流转输入，降低误操作。

组件结构：
1. 标题区：确认告警 / 标注误报 / 关闭告警。
2. 内容区：风险摘要 + remark 文本框。
3. 操作区：取消、确认提交。

交互规则：
1. 提交前二次确认。
2. 提交中禁用按钮防重复。
3. 成功后 toast 展示并刷新当前页面。

接口映射：
- `POST /alerts/{id}/confirm`
- `POST /alerts/{id}/false-positive`
- `POST /alerts/{id}/close`

---

### 5.6 统计分析页（趋势）
功能目标：查看疲劳与分心风险随时间变化，支持运营分析。

页面组件：
1. 筛选区：车队、时间范围、粒度（1m/5m/1h/1d）。
2. 主图表：疲劳分趋势线、分心分趋势线。
3. 辅助指标：峰值时间、平均值、波动区间。

交互流程：
1. 用户设置时间与粒度后请求数据。
2. 图表支持 hover 查看时间点明细。
3. 数据为空时展示“当前区间无数据”。

接口映射：
- `GET /stats/trend`

异常与边界：
1. 超长时间范围提醒用户调整粒度。
2. 接口失败时保留上次成功图并提示失败信息。

---

### 5.7 统计分析页（风险排行）
功能目标：识别高风险司机或车辆，支撑管理决策。

页面组件：
1. 维度切换：司机排行 / 车辆排行。
2. 条件区：时间范围、TopN。
3. 排行表：名次、对象ID、风险分、告警次数、环比变化。

交互流程：
1. 默认显示 Top10。
2. 切换维度后立即刷新。
3. 点击对象可跳转到对应告警筛选结果。

接口映射：
- `GET /stats/ranking`

异常与边界：
1. TopN 超过上限时前端限制输入。

---

### 5.8 规则管理页
功能目标：提供规则的可视化维护与生效控制。

页面组件：
1. 规则列表：规则编码、名称、阈值、持续时长、冷却时长、启用状态。
2. 新增规则弹窗。
3. 编辑规则弹窗。
4. 启停切换控件。

交互流程：
1. 页面加载查询规则列表。
2. ADMIN 可新增、编辑、启停。
3. 操作成功后刷新列表并提示审计已记录。

接口映射：
- `GET /rules`
- `POST /rules`
- `PUT /rules/{id}`
- `POST /rules/{id}/toggle`

异常与边界：
1. OPERATOR 与 VIEWER 全部禁用编辑动作。
2. 阈值输入需限制在 0~1。

### 5.9 审计与系统状态页
功能目标：给运营与管理人员提供可追溯操作视图和运行健康感知。

页面组件：
1. 审计日志表：操作人、操作类型、对象、时间、结果。
2. 系统状态卡：API 可用性、推送通道状态、关键延迟指标。
3. 时间筛选与关键字搜索。

交互流程：
1. 用户按时间窗查看操作记录。
2. 点击单条记录可展开详情（含 traceId）。
3. 系统状态定时刷新。

接口映射：

## 6. 页面状态设计
每个页面均必须包含以下状态：
1. 初始化加载态。
2. 数据成功态。
3. 空数据态。
4. 业务错误态（含错误码）。
5. 网络异常态（可重试）。

## 7. 字段展示映射（核心）
| 页面字段 | 来源接口字段 | 显示规则 |
|---|---|---|
| 告警编号 | `alertNo` | 表格主识别字段，支持复制 |
| 风险等级 | `riskLevel` | 1低2中3高，颜色标签区分 |
| 疲劳分 | `fatigueScore` | 保留2位小数 |
| 分心分 | `distractionScore` | 保留2位小数 |
| 触发时间 | `triggerTime` | 本地时区格式化显示 |
| 告警状态 | `status` | 0新建1确认2误报3关闭 |

## 9. 结论
DriveWeb 前端将以“实时总览 + 告警闭环 + 统计分析 + 规则管理”为主线构建界面功能。
后端接口用于驱动功能落地，前端设计聚焦可用性、可追溯与运营处理效率。
